Where's my view?

The following app has a single top-level view that fills the Display and draws a centered circle.

class SimpleCircle(display: Display): Application {
    init {
        display += view {
            size   = display.size
            render = {
                circle(Circle(
                    center = display.center,
                    radius = min(display.width, display.height) / 2 - 10
                ), fill = Red.paint)
            }
        }

        display.fill(White.paint)
    }

    override fun shutdown() {}
}

fun main() {
    application {
        SimpleCircle(display = instance())
    }
}

Where is my Button?​

Now we try the same thing with a [PushButton]. The following code feels like it should work; but it doesn't.

class InvisibleButton(display: Display): Application {
    init {
        display += PushButton().apply { size = Size(40, 20) }
        display.fill(White.paint)
    }

    override fun shutdown() {}
}

fun main() {
    application {
        InvisibleButton(display = instance())
    }
}
danger

The above examples does not render anything because PushButton delegates all rendering to its behavior and the button has none specified.

Setting a Behavior fixes the button​

The above example does not render anything because the button has no behavior installed. PushButton (like many Views in the controls library) does not render directly itself, but relies on its behavior for all drawing.

So we can fix the above app by explicitly adding a behavior to the button.

class ButtonWithBehavior(display: Display): Application {
    init {
        display += PushButton("Hi").apply {
            size = Size(80, 40)

            // Assign a Behavior to the button that handles rendering
            behavior = simpleButtonRenderer { button, canvas ->
                var color = Lightgray

                if (button.model.pressed    ) color = color.darker(0.25f)
                if (button.model.pointerOver) color = color.darker(0.25f)

                canvas.rect(rectangle = button.bounds.atOrigin, radius = 10.0, fill = color.paint)
            }

//            acceptsThemes = false // use along with themes to avoid the custom behavior being overridden
        }
        display.layout = constrain(display.first(), center)
        display.fill(White.paint)
    }

    override fun shutdown() {}
}

fun main() {
    // PointerModule required to support mouse/touch
    application(modules = listOf(PointerModule)) {
        ButtonWithBehavior(display = instance())
    }
}

A Theme could also be used​

Themes are a great way to assign behaviors to your entire app all at once. In this version we register the basicButtonBehavior module which makes a DynamicTheme available that will assign BasicButtonBehavior to all buttons in the app by default. This theme is injected into the app along with the ThemeManager which is used to select the theme. The result is that our button now has a behavior and delegates all rendering to it.

class ButtonWithBasicTheme(display: Display, themeManager: ThemeManager, theme: DynamicTheme): Application {
    init {
        // install theme that provides a button behavior,
        // thus automatically giving any button in the app a behavior
        themeManager.selected = theme

        display += PushButton().apply { size = Size(80, 40) }
        display.layout = constrain(display.first(), center)
        display.fill(White.paint)
    }

    override fun shutdown() {}
}

fun main() {
    // PointerModule required to support mouse/touch
    // nativeButtonBehavior provides a behavior for buttons (via dynamic theme)
    application(modules = listOf(PointerModule, nativeButtonBehavior())) {
        ButtonWithBasicTheme(display = instance(), themeManager = instance(), theme = instance())
    }
}
